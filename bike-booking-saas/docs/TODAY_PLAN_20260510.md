# Today Plan — bike-booking-saas

ไฟล์นี้คือแผนงานของวันนี้แบบ 4 เฟส สำหรับ `bike-booking-saas`

## เป้าหมายวันนี้

- ปิดหน้า `CraftBike Control Center` ให้ใช้คุมร้านได้ชัด
- วาง billing / expiry / due date ให้เป็นระบบ
- เตรียมทางไปสู่การสมัครร้านแบบ self-service
- ทำ safety / audit ให้พร้อมใช้งานจริง
- ทำงานต่อได้ต่อเนื่องแม้ต้องย้ายเครื่องไปอีกเครื่องระหว่างวัน

## กติกาการทำงานวันนี้

1. เริ่มจากอ่าน `START_HERE.md`
2. อ่าน `SESSION_NOTES_CURRENT.md` ก่อนแก้อะไร
3. ถ้าจะย้ายเครื่อง ให้หยุดที่จุดที่ repo สะอาดหรือรู้ชัดว่าค้างอะไรอยู่
4. ก่อนแก้ของเสี่ยง ต้อง backup ก่อนเสมอ
5. หลังจบแต่ละเฟส ให้สรุปสถานะสั้น ๆ และอัปเดต session notes
6. ถ้าจะส่งงานไปอีกเครื่อง ให้เปิด `START_HERE.md` แล้วตามด้วยไฟล์นี้

## Phase 1 — Control Center ให้คม

### เป้าหมาย

- ให้หน้า `control.craftbikelab.com` ใช้คุมร้านได้ชัดที่สุด
- ลบร้าน / suspend / reactivate / change status ใช้งานได้แน่น
- แสดงข้อมูลร้านสำคัญแบบเห็นแล้วตัดสินใจได้ทันที

### งานที่ต้องดู

- ตรวจว่าปุ่มลบร้านทำงานกับ Supabase live จริง
- เช็กสถานะร้านทุกแบบ
- ตรวจ summary บนหน้า control ว่าเห็นร้านที่ต้องดูด่วน
- จัดข้อมูลร้านให้ค้นหาได้ง่าย

### เสร็จเมื่อ

- control ใช้คุมร้านได้จริง
- ลบร้านไม่ติด permission
- สถานะร้านเปลี่ยนแล้วข้อมูลขึ้นตรง

## Phase 2 — Billing / Expiry / Due Date

### เป้าหมาย

- ให้ control มองเรื่องเงินและวันหมดอายุได้จริง
- รู้ว่าร้านไหนครบกำหนด ร้านไหนใกล้หมดอายุ ร้านไหนต้องพัก

### งานที่ต้องดู

- billing plan
- billing due date
- expiry date
- note การชำระเงิน
- ร้านที่ครบกำหนดใน 7 วัน
- ร้านที่หมดอายุแล้ว

### เสร็จเมื่อ

- มีข้อมูล billing ชัดใน control
- ร้านที่ due / expired ถูกมองเห็นง่าย
- พร้อมต่อยอดไปสู่การตั้งเตือนอัตโนมัติ

## Phase 3 — Self-service สมัครร้าน

### เป้าหมาย

- วาง flow สมัครร้านแบบเจ้าของร้านทำเองได้
- ใส่อีเมล จ่ายเงิน แล้วได้ลิงก์ใช้งาน 2 อัน

### Flow ที่ต้องรองรับ

1. ใส่อีเมล
2. เลือกแพ็กเกจ
3. จ่ายเงิน
4. สร้างร้าน
5. สร้าง owner account
6. ได้ลิงก์:
   - `[shop].craftbikelab.com`
   - `[shop]-admin.craftbikelab.com`

### เสร็จเมื่อ

- blueprint สมัครร้านพร้อม
- เห็นชัดว่าขั้นไหนต้องทำในแอป และขั้นไหนต้องทำใน Supabase / payment / deploy

## Phase 4 — Safety / Audit / Final Polish

### เป้าหมาย

- กันพลาดเวลาลบร้านหรือเปลี่ยนสถานะ
- เก็บประวัติการเปลี่ยนแปลงสำคัญ
- ทำให้ control ใช้งานจริงได้มั่นใจขึ้น

### งานที่ต้องดู

- confirm ตอนลบร้าน
- soft delete หรือ archive ถ้าจำเป็น
- audit trail การเปลี่ยนสถานะร้าน
- note ว่าใครทำอะไรกับร้านไหน

### เสร็จเมื่อ

- งานเสี่ยงมี safety ครบ
- รอบการทำงานวันนี้จบแบบเก็บต่อเครื่องอื่นได้

## ลำดับการทำงานที่แนะนำวันนี้

1. ปิด control ให้คม
2. ทำ billing / expiry / due date
3. วาง self-service สมัครร้าน
4. เก็บ safety / audit / polish

## ถ้าต้องย้ายเครื่องกลางวัน

ให้ทำตามนี้:

1. เช็กว่า repo สะอาดหรือรู้ว่ามีอะไรค้าง
2. `git status --short`
3. `git log --oneline -5`
4. อัปเดต session notes ถ้ามีสิ่งสำคัญเพิ่ม
5. commit / push ถ้าจบเป็นจุด
6. เปิด `START_HERE.md`
7. เปิดไฟล์นี้อีกครั้ง แล้วต่อจาก phase ล่าสุด
