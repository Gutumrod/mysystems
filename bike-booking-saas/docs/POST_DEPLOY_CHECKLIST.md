## 2. Pre-Check ก่อนกดเทส

### 2.1 Domain / SSL

- [ ] เปิดหน้า customer URL ได้
- [ ] เปิดหน้า admin URL ได้
- [ ] ใช้ `https://` ปกติ
- [ ] ไม่มีหน้า 403 / 404 / Domain misconfiguration

### 2.2 ข้อมูลร้าน

- [ ] ชื่อร้านแสดงถูกต้อง
- [ ] เบอร์โทรแสดงถูกต้อง
- [ ] LINE ID แสดงถูกต้อง
- [ ] Facebook แสดงถูกต้อง
- [ ] ไม่มี placeholder เช่น `ชื่อร้านจริงของคุณ`

### 2.3 บริการ

- [ ] มีรายการบริการแสดงในหน้า booking
- [ ] เรียงลำดับถูกต้อง
- [ ] ไม่มีบริการที่ควรซ่อนหลุดมา
- [ ] สามารถเลือกบริการได้อย่างน้อย 1 รายการ
- [ ] หน้าเว็บไม่ขึ้นข้อความแนว `ไม่มีตัวเลือกบริการ`

> ถ้า “ไม่มีบริการให้เลือก” ให้เช็กทันที:
> - ใน `bike_booking.service_items` มี row ของร้านนี้หรือไม่
> - `is_active = true` หรือไม่
> - `shop_id` ตรงกับร้านที่ deploy อยู่หรือไม่
> - service ถูกผูกกับ `NEXT_PUBLIC_SHOP_ID` ตัวเดียวกับหน้าเว็บหรือไม่
> - policy public read ของ `service_items` ยังอ่าน active rows ได้หรือไม่

### 2.4 ข้อมูลตั้งต้นในฐานข้อมูล

- [ ] มี shop row ของร้านที่กำลัง deploy จริง
- [ ] shop row มีเบอร์ / LINE / Facebook ครบ
- [ ] มี service_items อย่างน้อย 1 รายการ
- [ ] service_items ของร้านนี้เป็น `is_active = true`
- [ ] working hours ถูกต้อง
- [ ] วันหยุดพิเศษไม่บล็อกวันทดสอบโดยไม่ได้ตั้งใจ

## 3. Customer Flow

### 3.1 หน้าแรก

- [ ] หน้าโหลดได้ปกติ
- [ ] ไม่มี console error สำคัญ
- [ ] ปุ่ม/ช่องกรอกไม่เพี้ยนบน desktop
- [ ] ปุ่ม/ช่องกรอกไม่เพี้ยนบน mobile

### 3.2 ฟอร์มจอง

- [ ] กรอกชื่อได้
- [ ] กรอกเบอร์โทรได้
- [ ] กรอก Facebook/LINE ได้
- [ ] กรอกรุ่นรถได้
- [ ] กรอกปีรถได้
- [ ] เลือกบริการได้
- [ ] กรอกหมายเหตุได้
- [ ] เลือกวันที่ได้
- [ ] เลือกเวลาได้
- [ ] ปุ่ม `ยืนยันการจอง` กดได้เมื่อข้อมูลครบ
- [ ] ถ้าเลือกบริการไม่ได้ ระบบมีข้อมูลอธิบายสาเหตุชัดเจน

### 3.3 Validation

- [ ] ไม่กรอกชื่อ -> ระบบเตือน
- [ ] ไม่กรอกเบอร์ -> ระบบเตือน
- [ ] ไม่เลือกบริการ -> ระบบเตือน
- [ ] ไม่เลือกวันที่ -> ระบบเตือน
- [ ] ไม่เลือกเวลา -> ระบบเตือน
- [ ] ข้อความ error เป็นภาษาไทย

### 3.4 Submit

- [ ] จองสำเร็จ
- [ ] ได้ booking id
- [ ] redirect ไปหน้า success
- [ ] ไม่มี error จาก validation เรื่องบริการ ทั้งที่มีรายการบริการแสดงอยู่แล้ว

### 3.5 Success Page

- [ ] หน้า success เปิดได้
- [ ] เห็นข้อมูลการจองถูกต้อง
- [ ] ปุ่มคัดลอกใช้งานได้
- [ ] ข้อความที่คัดลอกครบ:
  - [ ] ชื่อร้าน
  - [ ] วันที่
  - [ ] เวลา
  - [ ] ชื่อลูกค้า
  - [ ] เบอร์โทร
  - [ ] รถ
  - [ ] บริการ
  - [ ] หมายเหตุ
  - [ ] รหัสการจอง

## 4. Admin Flow

### 4.1 Login

- [ ] เปิดหน้า login ได้
- [ ] login ได้
- [ ] ไม่โดนเด้งวนกลับหน้า login
- [ ] ถ้า user ไม่มีสิทธิ์ ระบบขึ้น unauthorized อย่างถูกต้อง
- [ ] user ที่ควรเข้าร้านนี้ได้ ไม่ถูกปฏิเสธผิดร้าน

### 4.2 Dashboard

- [ ] เข้าหน้า dashboard ได้
- [ ] เห็นรายการจองล่าสุด
- [ ] สถิติวันนี้/สัปดาห์นี้/เดือนนี้แสดงได้

### 4.3 Bookings

- [ ] booking ที่เพิ่งสร้างโผล่ในตาราง
- [ ] คลิกดูรายละเอียดได้
- [ ] เห็นชื่อ เบอร์ รถ บริการ หมายเหตุ ครบ

### 4.4 Status Update

- [ ] เปลี่ยนเป็น `in_progress` ได้
- [ ] เปลี่ยนเป็น `completed` ได้
- [ ] เปลี่ยนเป็น `cancelled` ได้
- [ ] เปลี่ยนเป็น `no_show` ได้
- [ ] ไม่มี toast error แปลกๆ

### 4.5 Calendar

- [ ] booking โผล่ใน calendar
- [ ] สี status ถูกต้อง
- [ ] คลิก event แล้วเห็นรายละเอียด

---

## 5. Database / Data Integrity

- [ ] booking row ถูกสร้างใน `bike_booking.bookings`
- [ ] `shop_id` ถูกต้อง
- [ ] `service_items` ถูกต้อง
- [ ] `booking_time_end` คำนวณถูกต้อง
- [ ] `customers` ถูกอัปเดตตาม booking
- [ ] no-show flow อัปเดต `no_show_count` ได้

---

## 6. Security / Hardening Check

- [ ] หน้า success ไม่อ่าน full booking row แบบ public policy เก่าแล้ว
- [ ] anonymous user ไม่อ่านข้อมูล booking เต็มแถวได้
- [ ] membership management จำกัด owner/staff ถูกต้อง
- [ ] RPC ที่จำเป็นยังทำงานได้
- [ ] ไม่มี regression หลัง hardening

---

- [ ] เห็นข้อมูลร้านจริง
- [ ] เห็นบริการอย่างน้อย 1 รายการ
- [ ] เลือกวัน/เวลาได้
- [ ] กดจองได้

### 7.2 Admin

- [ ] เปิดหน้า `booking-admin.craftbikelab.com/login` ได้
- [ ] login ได้ด้วยบัญชีที่ควรมีสิทธิ์
- [ ] เห็น booking ที่เพิ่งสร้าง
- [ ] เปลี่ยน status ได้อย่างน้อย 1 ครั้ง
