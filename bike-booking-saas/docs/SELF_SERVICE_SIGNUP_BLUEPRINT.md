# Self-Service Signup Blueprint

เอกสารนี้คือแนวทางสำหรับ phase ต่อไปของ CraftBike Platform:
เปิดให้คนสมัครร้านเอง ใส่อีเมล จ่ายเงิน แล้วได้ลิงก์ 2 อันอัตโนมัติ

## เป้าหมาย

- ลูกค้าใหม่สมัครร้านเองได้
- เจ้าของระบบไม่ต้องสร้างร้านมือทุกครั้ง
- สมัครเสร็จแล้วได้ลิงก์:
  - `https://[shop-slug].craftbikelab.com/`
  - `https://[shop-slug]-admin.craftbikelab.com/`
- ถ้าระบบกลางต้องคุมได้ ก็ยังเปิด `control.craftbikelab.com` ต่อไป

## Flow ที่ควรเป็น

1. ผู้ใช้กรอกอีเมล
2. เลือกชื่อร้าน / slug
3. เลือกแพ็กเกจ
4. ชำระเงิน
5. ระบบสร้างร้านใน `bike_booking.shops`
6. ระบบสร้าง owner account ใน `auth.users`
7. ระบบผูก owner เข้า `bike_booking.shop_users`
8. ระบบส่งลิงก์ลูกค้า + ลิงก์แอดมินให้ทันที

## ข้อมูลที่ต้องเก็บเพิ่ม

- `signup_requests`
  - เก็บอีเมลที่เริ่มสมัคร
  - เก็บ slug ที่ขอ
  - เก็บสถานะ signup
- `subscription_plans`
  - เก็บแพ็กเกจที่ขาย
  - เก็บราคา
  - เก็บอายุแพ็กเกจ
- `billing_invoices`
  - เก็บเลขบิล
  - เก็บสถานะการจ่าย
  - เก็บวันครบกำหนด
- `shop_billing_events`
  - ใช้เป็นประวัติการต่ออายุ / ปรับบิล / บันทึกจ่ายแล้ว

## สิ่งที่ทำต่อได้เลย

- วาง schema สำหรับ signup request
- วาง schema สำหรับ plan/invoice
- วางหน้า control สำหรับดูร้านสมัครใหม่
- วาง logic สร้าง slug และตรวจชนกัน
- วางหน้า signup public

## จุดที่ต้องรอเจ้าของระบบตัดสินใจ

- จะใช้ payment gateway อะไร
  - Stripe
  - โอนเงิน/manual payment
  - หรือเริ่มจาก manual approval ก่อน
- จะให้ auto-activate หลังจ่ายหรือให้คนกดอนุมัติ
- จะให้ trial กี่วัน
- จะขายแพ็กเกจรายเดือนหรือรายปีเป็นหลัก

## ข้อสรุป

ส่วนนี้ยังเริ่มทำโครงได้ทันที
แต่จุดชำระเงินจริงต้องรอเลือก payment flow ก่อน
