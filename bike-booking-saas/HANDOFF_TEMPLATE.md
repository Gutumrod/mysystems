# Handoff Template

ใช้ไฟล์นี้เวลาส่งงานระหว่างเครื่อง หรือส่งต่อให้ Codex/Claude/Gemini อ่านแล้วทำต่อได้ทันที

## เริ่มต้น

```text
โปรเจกต์: bike-booking-saas
ที่อยู่โปรเจกต์: [path]

ให้อ่านก่อน:
1. START_HERE.md
2. docs/WORKSPACE_RULES.md
3. docs/WORKFLOW.md
4. SESSION_NOTES_CURRENT.md
5. docs/DOMAIN_STANDARD.md
6. README.md

ขอบเขตงาน:
- ทำเฉพาะ bike-booking-saas
- อย่าแตะ Chatbot ถ้าไม่ได้สั่ง

สถานะล่าสุด:
- [สรุปสั้น ๆ ว่าทำอะไรไปแล้ว]
- [commit ล่าสุด]
- [สิ่งที่ยังค้าง]

งานต่อไป:
1. [งานข้อ 1]
2. [งานข้อ 2]
3. [งานข้อ 3]

ข้อควรระวัง:
- [เช่น backup ก่อนแก้ schema]
- [เช่น ต้อง pull main ก่อน]

ไฟล์ที่เกี่ยว:
- [path 1]
- [path 2]
- [path 3]
```

## ตอนจบงาน

```text
เสร็จแล้วให้รายงานกลับมาแบบสั้น ๆ ว่า:
- ทำอะไรเสร็จแล้ว
- repo status เป็นยังไง
- commit / push หรือยัง
- ต้องทำอะไรต่อ
- ถ้าย้ายเครื่อง ให้เปิดไฟล์ไหนต่อ
```

## เวอร์ชันสั้นมาก

```text
อ่าน START_HERE.md, docs/WORKSPACE_RULES.md, docs/WORKFLOW.md, SESSION_NOTES_CURRENT.md, docs/DOMAIN_STANDARD.md, README.md ก่อน
ทำเฉพาะ bike-booking-saas
ทำเสร็จสรุปสั้น ๆ แล้ว commit / push
```
